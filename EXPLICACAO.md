# Explicação sobre o Reenvio das Atividades

Durante o desenvolvimento das atividades de Programação Orientada a Objetos (POO), ocorreu um problema de configuração com duas contas Git cadastradas na mesma máquina (`Fabricio-Nascimentto` e `Fabricio-Nascimentto2`), o que acabou gerando conflito e exclusão acidental do histórico de commits do repositório original.

Para corrigir a situação, o repositório foi recriado e as atividades foram reenviadas utilizando as datas originais de entrega (através da flag `--date` no Git), preservando assim a fidelidade cronológica e o conteúdo de cada entrega.

Nenhuma atividade foi modificada, apenas reenviada conforme os registros originais.

Este documento tem o objetivo de esclarecer o ocorrido e garantir a transparência do processo de recuperação do histórico.
